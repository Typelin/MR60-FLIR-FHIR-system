# 前後端分離架構與重要性說明書

本文件詳細闡述了 **FLIR E53 熱成像測溫與 FHIR 醫療資訊系統** 採用「前後端分離」架構的設計理念、運作機制及其對於本專案的核心價值，以供專案報告與系統設計參考。

---

## 1. 系統架構圖 (System Architecture)

本專案採用現代網頁開發標準的「雲端前端 + 地端後端」混合型前後端分離架構：

```mermaid
graph TD
    subgraph 雲端應用層 (Cloud Front-end)
        A[使用者瀏覽器] <-->|HTTPS 載入網頁| B(Cloudflare Pages)
    end

    subgraph 通訊與穿牆橋樑 (Tunnel)
        A <-->|發送 API 請求 GET/POST| C(Cloudflare Tunnel)
    end

    subgraph 地端控制與服務層 (Local Back-end)
        C <-->|安全映射 Port 8080| D(學校電腦: Python API Server)
        D <-->|原始測溫數據上傳| E(HAPI FHIR / PostgreSQL 資料庫)
        F[FLIR E53 熱成像儀] -->|Type-C 數據讀取| G(Python 測溫控制程式)
        G -->|自動組裝 FHIR JSON| D
    end
```

## 2. 為什麼採用前後端分離？（四大核心價值）

對於在學校環境中開發 IoT 與醫療整合專案，前後端分離具備以下不可替代的優勢：

### ① 突破校園網路限制與防火牆封鎖
*   **痛點**：學校電腦處於嚴格的校園內網（NAT），沒有實體公網 IP，且網管通常禁止進行 Port 轉向（Port Forwarding）。若將網頁與伺服器混在一起架設，校外人士（如指導教授在家中、評審委員）將無法連上系統。
*   **解法**：將前端網頁靜態檔案（HTML/CSS/JS）託管於 **Cloudflare Pages**（全球 CDN 節點，隨時可秒速開啟）。後端資料庫與 API 留在學校電腦，透過 **Cloudflare Tunnel** 主動連出建立隧道。如此一來，任何人只要在瀏覽器輸入網址，網頁就能流暢讀寫學校電腦的資料。

### ② 提升系統資安與醫療隱私防護
*   **痛點**：前端網頁如果直接連線去讀取 SQL 資料庫，會將資料庫帳號密碼暴露在瀏覽器的原始碼中，且資料庫通訊埠暴露在公網極易遭受駭客攻擊。
*   **解法**：前端網頁完全不接觸資料庫，僅透過標準的 **HTTPS RESTful API** 與後端通訊。後端 API 伺服器負責驗證請求，再代為讀寫 PostgreSQL，確保病人隱私資料不外洩。

### ③ 提升團隊協作與並行開發效率
*   **痛點**：若前後端代碼混雜在一起，每當修改網頁畫面時，就必須重啟或變動後端的 C SDK 與相機連接邏輯，極易造成系統崩潰。
*   **解法**：
    *   **前端開發者**：只需專注於 React/Vue/HTML 網頁與美觀的圖表設計。他不需要安裝 FLIR 相機驅動，也不需要手邊有實體相機，只要將 API 網址指向學校電腦的穿牆網址，即可在自己筆電上進行開發。
    *   **後端與硬體開發者**：可以專心在學校電腦調整 Python 程式、對接 C SDK 讀取鏡頭溫度，並設計資料庫結構，兩端開發完全互不干擾。

### ④ 符開業界標準，成果展示大幅加分
*   **專題加分項**：本架構完全符合現代大型醫療資訊系統的雲端部署模式。在成果發表時，說明系統採用了「**前後端分離、地端設備對接、雲端靜態託管與安全隧道穿牆**」的架構，能向評審委員證明系統具備極高的實用性與技術成熟度，是極佳的加分項目。

## 3. 開發與部署分工對照表

| 系統層級 | 部署平台 | 使用技術 | 負責功能 |
| :--- | :--- | :--- | :--- |
| **前端 (Frontend)** | Cloudflare Pages | HTML5 / CSS3 / JavaScript (或 React) | • 使用者互動介面<br>• 即時體溫折線圖與儀表板<br>• 病人登錄與查詢表單 |
| **通訊橋樑 (Tunnel)** | Cloudflare Network | cloudflared 穿牆隧道 | • 將地端 `localhost:8080` 安全映射至公網 HTTPS 網址 |
| **後端 (Backend)** | 學校主機 (地端) | Python (server.py) / Java / PostgreSQL | • 接收前端 API 請求 (GET/POST)<br>• 自動處理 FLIR 鏡頭體溫封裝<br>• 寫入/讀取標準 FHIR 醫療資料 |
