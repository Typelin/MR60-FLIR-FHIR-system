# 專案當前開發進度說明書

**文件日期：** 2026 年 7 月 9 日

本文件記錄了 **FLIR E53 熱成像測溫與 FHIR 醫療系統整合專案** 目前在學校主機端的測試現狀、已驗證功能，以及後續待實現的開發任務。

---

## 1. 當前已完成測試項目 (已驗證)

*   **[✓] 外網穿透連線測試 (Cloudflare Tunnel)**：
    *   已成功部署並啟動模擬後端伺服器（Port 8080）。
    *   成功透過 `cloudflared.exe` 建立隧道，將流量引流至台北機房。
    *   已驗證使用校外個人電腦與手機，能透過 `https://helen-pavilion-expensive-jewel.trycloudflare.com` 公網網址成功讀取學校主機的資料。
*   **[✓] C SDK 動態連結庫 (DLL) 載入測試**：
    *   已驗證 Python 3.12 能使用 `ctypes` 成功載入 `atlas_c_sdk.dll`。
    *   修正了官方錯誤訊息讀取的 Bug，確保在無錯誤時不會誤判崩潰。
*   **[✓] 靜態熱影像資料讀取 (模擬)**：
    *   `test_rjpeg.py` 能成功以相對路徑對接 SDK，具備開啟 RJPEG 圖片並提取全圖溫度矩陣（NumPy Array）的程式架構。
*   **[✓] 無鏡頭即時畫面渲染模擬器**：
    *   `flir_demo_emulator.py` 能在無相機時以 OpenCV 渲染動態熱源、追蹤最高溫十字，並模擬高溫警報與滑鼠即時測溫。

---

## 2. 當前未實現項目 (待下星期實體相機到貨後測試)

*   **[ ] 實體 E53 相機即時串流測試**：
    *   待使用 USB Type-C 線連接 E53 相機與學校電腦。
    *   執行 `test_live_stream.py` 驗證是否能透過 `ACS_Discovery_scan` 自動偵測到 USB 相機並開啟影像串流，並在終端機輸出即時溫度。
*   **[ ] 本地 FHIR 伺服器 (HAPI FHIR) 部署**：
    *   因學校電腦群組原則（Group Policy）鎖死，導致無法升級 WSL，因此**無法使用 Docker 部署**。
    *   **替代方案**：待手動安裝 Java JDK 17 與 Maven，在學校電腦原生啟動 HAPI FHIR 伺服器，或連線至外部免費的 FHIR 雲端測試資料庫。
*   **[ ] 熱成像數據與 FHIR 伺服器對接**：
    *   待撰寫 `upload_to_fhir.py`，將 E53 實體鏡頭讀取到的即時體溫，自動封裝成 `Observation` (Vital Signs) 的符合衛福部標準的 JSON 格式，並以 HTTP POST 方式上傳至 FHIR 伺服器。
*   **[ ] 前端網頁（React/Vue）串接與視覺化**：
    *   待撰寫前端網頁，透過 Cloudflare Tunnel 讀取 FHIR 資料庫，並將溫度歷史紀錄、病人清單渲染成漂亮的圖表網頁。
